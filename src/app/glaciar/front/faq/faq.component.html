F.A.Q.

Preguntas Frecuentes



¿Que es "Glaciar"?

Es una plataforma que realiza la visualización de la calidad del aire y del agua, sobre una serie de dataset publicados en cinco plataformas de datos abiertos.


¿Por qué se llama "Glaciar"?

El nombre "Glaciar" es simbólico, y hace referencia a la capacidad que tienen los glaciares de mantener en su interior información histórica de la calidad del aire y del agua a lo largo de los siglos.



Desarrolladores de Software

Preguntas Frecuentes


¿El código fuente de la aplicación está disponible?

Si, el código de la plataforma "Glaciar" está disponible en el repositorio de GitHub     glaciar-org.


¿Que componentes de la aplicación se comparten en GitHub?

Se comparten los tres componentes principales:

  • El front-end, escrito en Angular
  • El back-end, escrito en Node.js y express
  • Los procesos ETL (extracción, transformación y carga) que extraen los datos de las plataforma de datos abiertos

¿La plataforma "Glaciar" se trata de un proyecto open source en donde puedo colaborar?

Si! los pull request son bienvenidos!


¿La plataforma "Glaciar" se trata de un proyecto open source que se puede extender?

Si, se puede extender y/o modificar el software de la plataforma "Glaciar" siempre que se respeten las licencias del software de terceros.


¿Que software de terceros la plataforma "Glaciar" utiliza bajo licencia?

La plataforma Glaciar está construida utilizando productos open source, y la librería comercial amChart4 bajo su licencia gratuita que permite su uso, incluso dentro de proyectos open source, si se mantiene un pequeño logo de atribución a amChart visible en el gráfico (amcharts store)


¿Puedo conocer con que tecnologías se contruyó la platforma "Glaciar"?

Si, a continuacíon se enumeran los proyectos utilizados:

  • angular.io
  • nodejs.org
  • Express
  • Bootstrap
  • ng-bootstrap
  • ngx-admin
  • amcharts
